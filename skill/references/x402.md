# x402 handshake — exact shapes

The premium brief is metered with the [x402](https://x402.org) pattern: the
payment IS the authentication. No account, no key.

## 1. Unpaid request → HTTP 402

`GET /premium/brief/{id}` (or the `get_premium_brief` MCP tool with no
`payment_token`) returns the payment contract:

```json
{
  "x402Version": 1,
  "error": "Payment required",
  "accepts": [{
    "scheme": "exact",
    "network": "eip155:1439",
    "asset": "USDC",
    "maxAmountRequired": "5000",
    "payTo": "0x0000000000000000000000000000000000000000",
    "resource": "/premium/brief/{id}",
    "description": "SIDELINE premium match brief — pay-per-call, no account, no key",
    "mimeType": "application/json"
  }]
}
```

- `maxAmountRequired` is in USDC base units (6 decimals): `5000` = `0.005 USDC`.
- `network` `eip155:1439` targets the Injective EVM testnet.
- `payTo` and the price come from the server's `PAY_TO` / `PRICE_UNITS` env.

## 2. Paid request → HTTP 200 + token

Retry with an `X-PAYMENT` authorization header (an EIP-3009 transfer
authorization in production; any non-empty value in demo mode):

```
GET /premium/brief/{id}
X-PAYMENT: <signed EIP-3009 authorization>
```

Response body includes `paid: true` and a `payment_token`; the same settlement
receipt is echoed in the response header:

```
X-PAYMENT-RESPONSE: {"settled":true,"token":"<hex>"}
```

## 3. Reuse the token against MCP

Pass `payment_token` to the `get_premium_brief` MCP tool to unlock the brief
from an AI client. Tokens are short-lived (15 min in the reference server).

## Production vs demo

- **Demo mode** accepts the handshake and issues a token immediately so the
  end-to-end loop is demonstrable with zero on-chain setup.
- **Production** submits the `X-PAYMENT` EIP-3009 authorization to a facilitator
  / relayer for on-chain USDC settlement before serving — the marked
  integration point in `src/server.ts`.
